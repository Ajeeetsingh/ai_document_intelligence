# Cursor Project Rules

## General

- Follow existing architecture.
- Do not introduce new patterns without approval.
- Do not add dependencies without approval.
- Do not modify database schema without approval.
- Do not modify API contracts without approval.

## Development

- Generate implementation plan before coding.
- Wait for approval before implementation.
- Follow feature specifications exactly.

## Code Quality

- Keep business logic out of routes.
- Keep controllers thin.
- Use services for business logic.
- Use type hints.
- Write docstrings.

## Documentation

- Update PROJECT_STATE.md after feature completion.
- Update feature documentation when behavior changes.
